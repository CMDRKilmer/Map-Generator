## 概述

为项目搭建自动化 CI/CD 工作流，引入 `ruff` 作为 Python 代码规范和格式化工具，配置 GitHub Actions 在每次推送和 PR 时自动执行 lint 检查和测试。

## 新增文件

| 文件 | 说明 |
|------|------|
| `.github/workflows/ci.yml` | GitHub Actions CI 工作流 — `lint` (ruff check + format) + `test` (pytest) |
| `pyproject.toml` | ruff 规则配置 (E/F/W/I)、pytest 路径配置 |

## CI 工作流（`.github/workflows/ci.yml`）

- **触发条件**: 推送到 `main` 分支、针对 `main` 的 Pull Request
- **作业**:
  - **lint**: 安装 ruff → `ruff check .` → `ruff format --check .`
  - **test**: 安装系统依赖 (`libegl1`, `libgl1`, `libxkbcommon0` 等) → `pip install -r requirements.txt pytest` → `QT_QPA_PLATFORM=offscreen python3 -m pytest tests/ -v`
- **Python 版本**: 3.11

## 代码格式清理

使用 `ruff format` 对全项目 10 个 Python 文件（`core/*.py`, `export/exporter.py`, `main.py`, `ui/*.py`, `utils/colors.py`, `tests/*.py`）进行了格式化并移除了 37 个 lint 错误。

## 本地验证

```bash
# lint
ruff check .
# format check
ruff format --check .
# test
QT_QPA_PLATFORM=offscreen python3 -m pytest tests/ -v
```

三者全部通过 ✅
